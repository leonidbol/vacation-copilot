# Agent Spec: Vacation Copilot

## Overview
An elite, proactive travel concierge agent designed to minimize cognitive load for travelers. Rather than managing complex multi-agent routing networks, it handles two core behavioral modes contextually within a streamlined **Unified Flat Agent Architecture**: interactive pre-trip onboarding and active on-the-ground dining recommendations.

## Example Use Cases

### Onboarding
- **Input**: "I'm going to Barcelona Aug 15-22. Here's my flight: Delta DL456..."
- **Output**: Agent extracts flight details, invokes `parse_travel_document`, saves to itinerary, and contextually prompts for hotel information.

### Menu Matching
- **Input**: User pastes an unstructured restaurant menu in a foreign language.
- **Output**: Agent interprets the list, cross-references the saved traveler profile, and maps top dish recommendations directly to dietary requirements.

## Tools Required

| Tool | Purpose |
| :--- | :--- |
| `parse_travel_document` | Extract structured data tokens from unstructured travel text strings. |
| `save_flight` / `save_hotel` / `save_activity` | Persist extracted trip logistics as local Markdown documentation. |
| `save_user_profile` / `get_user_profile` | Manage traveler-specific dietary profiles, restrictions, and preferences. |
| `parse_and_match_menu` | Analyze unstructured culinary menus against the traveler's active profile. |
| `save_menu_recommendations` | Persist curated restaurant recommendations to the user space. |
| `get_itinerary` | Retrieve compiled trip data blocks for runtime validation. |

## Constraints & Safety Rules
- **Financial Boundary**: Never book, reserve, or charge anything on behalf of the user.
- **Verification Loop**: Always confirm extracted structural data with the user before committing saves.
- **Dietary Liability**: Do not provide medical or clinical dietary advice. 
- **Allergen Protocol**: Clearly flag critical allergen warnings with a mandatory explicit disclosure: *"Please confirm cross-contamination risks directly with restaurant staff."*

## Success Criteria
1. Onboarding workflow collects and resolves all primary trip logistics within 4–5 conversational turns.
2. Menu recommendation engine successfully catches and flags implicit or explicit dietary violations.
3. All state data persists cleanly as human-readable, portable Markdown files.
4. The system routes seamlessly between onboarding prompts and menu lookups based entirely on conversational context flags.
